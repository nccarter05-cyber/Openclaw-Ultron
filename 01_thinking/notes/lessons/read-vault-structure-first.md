# Read Vault Structure First

## The Lesson
When organizing new information (ideas, lessons, notes), always check the vault structure **before** placing it. Don't default to MEMORY.md as a catch-all.

The schema exists for a reason:
- **Ideas** → `01_thinking/notes/ideas/`
- **Lessons** → `01_thinking/notes/lessons/`
- **Claims** → `01_thinking/notes/claims/`
- **Long-term memory** → `MEMORY.md` (curated, session-independent facts)

## Why This Matters
Defaulting to MEMORY.md wastes round trips and defeats the organizational system. Proper placement means:
- Information is findable later
- The right tool for each job gets used
- No rework needed
